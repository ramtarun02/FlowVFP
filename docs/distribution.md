# FlowVFP Local Distribution Build

This process creates a self-contained Windows distribution for FlowVFP with one LAN URL for frontend, API, and Socket.IO.

## What it produces

1. Portable runtime folder: VFP-Python/dist/FlowVFP
2. Installer executable (if Inno Setup is installed): VFP-Python/dist/installer/FlowVFP-Setup-<version>.exe

## End-user requirements mapping

1. Setup file installs all dependencies/scripts:
   The installer ships the full PyInstaller runtime bundle and solver tools.
2. Frontend calls the right URL:
   Packaged frontend uses same-origin mode (VITE_PACKAGED=true), so API calls resolve to the running launcher URL.
3. HTTP and Socket behind one LAN URL:
   launcher.py serves SPA static files and backend/socket service on one host:port.
4. Self-contained runtime with no host Python requirement:
   PyInstaller bundle contains Python runtime and app dependencies.

## Build command

From workspace root:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\build_flowvfp_dist.ps1
```

To build only portable output:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\build_flowvfp_dist.ps1 -SkipInstaller
```

## Runtime config for first run

Launcher creates config at:

- %PROGRAMDATA%\FlowVFP\runtime-config.json

Default config:

```json
{
  "host": "0.0.0.0",
  "port": 5000,
  "open_browser": true,
  "secret_key": "<auto-generated>"
}
```

Adjust host/port for your LAN policy.

## Notes

- Existing cloud and development flows are unchanged.
- Packaged launcher expects frontend assets under VFP-Python/frontend_build during build.
- Installer generation requires Inno Setup 6 (ISCC.exe).
